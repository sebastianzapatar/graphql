Guia de Despliegue: Render + Netlify

Backend en Render • Frontend en Netlify • Base de datos en MongoDB Atlas

Arquitectura de Despliegue

En produccion separamos los servicios:

Backend (Render)

Spring Boot + GraphQL desplegado como Web Service en Render. Se conecta a MongoDB Atlas.

Frontend (Netlify)

React + Vite desplegado como sitio estatico en Netlify. Se conecta al backend en Render.

Paso 1: Configurar MongoDB Atlas (Base de Datos)

Necesitamos una base de datos MongoDB en la nube (gratis).

1Ir a MongoDB Atlas y crear una cuenta gratuita.

2Crear un Cluster gratuito (M0 Free Tier).

3En Database Access, crear un usuario con password.

4En Network Access, agregar 0.0.0.0/0 para permitir acceso desde cualquier IP (necesario para Render).

5Ir a Connect → Drivers y copiar la URI de conexion:

mongodb+srv://USUARIO:PASSWORD@cluster0.xxxxx.mongodb.net/graphql_eia?retryWrites=true&w=majority
Importante: Reemplaza USUARIO y PASSWORD con tus credenciales reales. Guarda esta URI, la necesitaras en Render.

Paso 2: Desplegar Backend en Render

1Ir a render.com y crear una cuenta (puedes usar GitHub).

2Click en "New" → "Web Service".

3Conectar tu repositorio de GitHub donde esta el proyecto.

4Configurar el servicio:

Name:           graphql-backend
Root Directory: ejemplo-graphql
Runtime:        Docker
Branch:         main

5En "Environment Variables", agregar:

SPRING_DATA_MONGODB_URI = mongodb+srv://USUARIO:PASSWORD@cluster0.xxxxx.mongodb.net/graphql_eia

6Click en "Create Web Service". Render construira la imagen Docker automaticamente.

7Esperar el deploy (~3-5 min). Render asignara una URL:

https://graphql-backend-xxxx.onrender.com
Verificar: Abrir https://TU-URL.onrender.com/graphiql para confirmar que GraphiQL funciona.

Paso 3: Desplegar Frontend en Netlify

1Ir a netlify.com y crear una cuenta.

2Click en "Add new site" → "Import an existing project".

3Conectar el repositorio de GitHub.

4Configurar el build:

Base directory:    ejemplo-graphql/frontend
Build command:     npm run build
Publish directory: ejemplo-graphql/frontend/dist

5En "Environment variables", agregar:

VITE_GRAPHQL_URL = https://graphql-backend-xxxx.onrender.com/graphql

Reemplaza con la URL real de tu backend en Render.

6Click en "Deploy site". Netlify construira y desplegara automaticamente.

7Netlify asignara una URL:

https://graphql-productos-xxx.netlify.app
Redirecciones SPA: Crear un archivo public/_redirects en el frontend con:

/* /index.html 200

Esto permite que React Router funcione correctamente en Netlify.

Paso 4: Verificar el Despliegue

1Abrir el frontend en Netlify y verificar que carga correctamente.

2Crear un producto usando el formulario.

3Verificar que aparece en la lista de productos.

4Abrir GraphiQL en Render y ejecutar una query para confirmar que los datos estan en MongoDB Atlas.

Troubleshooting

El frontend no conecta con el backend

  • Verificar que VITE_GRAPHQL_URL apunta a la URL correcta de Render.
  • Verificar que CORS esta habilitado en el backend (ya esta en application.yml).
  • Redesplegar el frontend despues de cambiar variables de entorno.

El backend no conecta con MongoDB Atlas

  • Verificar la URI en SPRING_DATA_MONGODB_URI (usuario, password, nombre del cluster).
  • Verificar que 0.0.0.0/0 esta en Network Access de Atlas.
  • Verificar los logs en Render (Dashboard → Logs).

Render tarda mucho o no responde

  • El tier gratuito de Render tiene cold starts (~30s si el servicio estuvo inactivo).
  • Considerar un tier pago si necesitas respuestas rapidas.
