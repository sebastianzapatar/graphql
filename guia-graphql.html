Guia: Como Probar GraphQL

Universidad EIA • Ingenieria de Sistemas • Proyecto GraphQL + MongoDB + Java 21

1. Requisitos Previos

1Java 21 instalado. Verificar con:

java --version
# Debe mostrar: openjdk 21.x.x o superior

2Docker instalado y corriendo. Verificar con:

docker --version
docker compose version

3Maven instalado o usar el wrapper del proyecto.

2. Levantar MongoDB con Docker

Desde la carpeta ejemplo-graphql/:

cd ejemplo-graphql
docker compose up -d

Verificar que el contenedor esta corriendo:

docker ps
# Debe mostrar: mongo-graphql corriendo en puerto 27017
Tip: Si ya tienes MongoDB corriendo en Docker, asegurate de que el puerto 27017 no este ocupado.

3. Ejecutar la Aplicacion

mvn spring-boot:run

Esperar hasta ver en consola:

Started GraphqlApplication in X.XXX seconds

4. Abrir GraphiQL

Abrir en el navegador:

http://localhost:8080/graphiql

GraphiQL es una interfaz visual para escribir y ejecutar consultas GraphQL. Tiene autocompletado y documentacion integrada.

5. Operaciones: Queries (Lectura)

5.1 Obtener todos los productos

query {
  productos {
    id
    nombre
    descripcion
    precio
    stock
    categoria
  }
}

5.2 Buscar producto por ID

query {
  productoPorId(id: "PEGAR_ID_AQUI") {
    nombre
    precio
    stock
  }
}

5.3 Buscar por categoria

query {
  productosPorCategoria(categoria: "Electronica") {
    nombre
    precio
  }
}

5.4 Buscar por rango de precio

query {
  productosPorPrecio(min: 10000, max: 5000000) {
    nombre
    precio
    categoria
  }
}
Tip: Puedes pedir SOLO los campos que necesites. No estas obligado a pedir todos.

6. Operaciones: Mutations (Escritura)

6.1 Crear un producto

mutation {
  crearProducto(
    nombre: "Laptop HP Pavilion"
    descripcion: "16GB RAM, 512GB SSD, Intel i7"
    precio: 3500000
    stock: 15
    categoria: "Electronica"
  ) {
    id
    nombre
    precio
  }
}
Importante: Copia el id que retorna para usarlo en las siguientes operaciones.

6.2 Actualizar un producto

mutation {
  actualizarProducto(
    id: "PEGAR_ID_AQUI"
    precio: 2999000
    stock: 8
  ) {
    id
    nombre
    precio
    stock
  }
}

6.3 Eliminar un producto

mutation {
  eliminarProducto(id: "PEGAR_ID_AQUI")
}

Retorna true si fue eliminado, false si no existia.

7. Usando Variables (Avanzado)

En la seccion "Variables" de GraphiQL (panel inferior):

Query:

query BuscarPorPrecio($min: Float!, $max: Float!) {
  productosPorPrecio(min: $min, max: $max) {
    nombre
    precio
    categoria
  }
}

Variables (JSON):

{
  "min": 50000,
  "max": 2000000
}

8. Probar con cURL (Terminal)

# Obtener todos los productos
curl -X POST http://localhost:8080/graphql \
  -H "Content-Type: application/json" \
  -d '{"query": "{ productos { id nombre precio } }"}'

# Crear un producto
curl -X POST http://localhost:8080/graphql \
  -H "Content-Type: application/json" \
  -d '{"query": "mutation { crearProducto(nombre: \"Teclado\", precio: 120000, stock: 50, categoria: \"Perifericos\") { id nombre } }"}'

9. Explorar el Schema

En GraphiQL, haz clic en el icono de "Docs" (esquina superior derecha) para ver:

  • Todos los tipos definidos (Producto)
  • Todas las queries disponibles
  • Todas las mutations disponibles
  • Tipos de cada campo (String!, Float!, Int!, etc.)
Tip: GraphQL es auto-documentado. El schema ES la documentacion. No necesitas Swagger.

10. Detener los Servicios

# Detener Spring Boot: Ctrl+C en la terminal

# Detener MongoDB
docker compose down

# Detener y borrar datos
docker compose down -v
